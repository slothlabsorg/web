Tired of "network service terminated"?

Leapp, Granted, and other Electron-based AWS tools use Chromium's own TLS stack — which ignores your system certificate store. If your company uses Cloudflare Zero Trust, a corporate CA, or any SSL-inspecting proxy, those apps simply crash or refuse to connect.

CloudSwitch is built on Tauri, which uses WKWebView on macOS. WKWebView trusts the exact same certificates as Safari and your system browser — so corporate VPN, Cloudflare Gateway, or custom CA certificates work out of the box. No configuration needed.

Features

Everything you need to work with AWS.
Nothing you don't.

CloudSwitch covers the full AWS developer workflow — from SSO login to terminal sessions inside EC2 instances.

SSO Login

Browser-based OIDC device flow. The browser opens on your system, so corporate CAs are always trusted. SSO tokens go into the macOS Keychain and ~/.aws/sso/cache/ (compatible with AWS CLI).

Multi-Account Roles

Assume any role across all your accounts with one click. Credentials are written to ~/.aws/credentials in the standard AWS CLI format — every tool just works.

EKS Discovery

Auto-discover all EKS clusters in the active region. One click configures ~/.kube/config with the correct exec credential plugin — no manual aws eks update-kubeconfig.

AWS Web Console

Open the AWS Management Console already signed in as the assumed role. Uses the federation sign-in endpoint — no re-authentication needed, and it opens the correct regional console.

EC2 via SSM

List all EC2 instances and connect to any running instance via AWS Systems Manager Session Manager. No SSH keys, no bastion hosts, no open ports. Opens in your default terminal (Terminal.app or iTerm2).

Copy Credentials

Copy the current role's credentials as shell export statements in one click. Paste into any terminal to set AWS_ACCESS_KEY_ID, SECRET_ACCESS_KEY, and SESSION_TOKEN instantly.

Security

Your credentials. Truly yours.

Everything stays on your machine. No cloud sync, no telemetry, no third-party servers.

🔐

System Vault

SSO access tokens are stored in the macOS Keychain (Windows: Credential Vault, Linux: Libsecret) via the keyring crate. Tokens are encrypted at rest using the OS's own crypto — the same protection used by your browser's saved passwords.

🌐

Direct to AWS

All API calls go directly from the Rust backend to AWS endpoints. There is no proxy, no relay, and no analytics service. The app has no network calls outside of *.amazonaws.com and signin.aws.amazon.com.

🔗

CLI Compatible

Credentials are written to ~/.aws/credentials and SSO tokens to ~/.aws/sso/cache/ in the exact format the AWS CLI uses. Terraform, kubectl, Pulumi — every tool picks them up automatically.

credentials.rs 
// Credentials own their data — the compiler
// guarantees they can't outlive the session.
#[derive(Serialize)]
pub struct Credentials {
    pub access_key_id:     String,
    pub secret_access_key: String,
    pub session_token:     String,
    pub expires_at: Option<String>,
}

// ? propagates errors — no try/catch,
// no uncaught exceptions, no silent failures.
pub async fn assume_role(…) -> Result<Credentials, String> {
    let token = read_cached_token(&start_url)
        .ok_or("Not logged in")?;
    let resp  = sso.get_role_credentials()
        .send().await
        .map_err(|e| e.to_string())?;
    Ok(Credentials { … })
}
Roadmap

Plugin architecture — coming soon

The goal is a lightweight, shell-first plugin system. No JavaScript bundles, no npm, no build steps — just scripts and config.

🪝

Lifecycle Hooks

Shell scripts that fire on pre-login, post-assume-role, on-expiry events. Trigger Slack notifications, refresh kubeconfigs, update .envrc files automatically.

🔌

Credential Types

Add new credential sources beyond SSO: IAM User access keys, cross-account chains, OIDC providers (Okta, Azure AD, Google Workspace). Each type is a Rust trait implementation.

📊

Dashboard Panels

Embed custom WebView panels into the main content area. Build a cost dashboard, a CloudWatch widget, or a custom resource viewer — all from a single URL.

~/.config/aws-switch/plugins.toml 
# Notify Slack when a role is assumed
[[plugin]]
name    = "slack-notify"
type    = "hook"
on      = "post-assume-role"
command = "~/.aws-switch/slack.sh"
args    = ["--channel", "#aws-access"]

# Auto-refresh .envrc in current project
[[plugin]]
name    = "direnv-refresh"
type    = "hook"
on      = "post-assume-role"
command = "direnv allow ."

# Custom cost dashboard panel
[[plugin]]
name  = "cost-panel"
type  = "panel"
url   = "http://localhost:8080/costs"
icon  = "💰"
title = "Cost Explorer"
